# Experiment Workflow

The experimental workflow follows six stages:

1. Prepare low-fidelity sketches for each scenario.
2. Upload sketches to the model-specific API script.
3. Run Phase 1 to generate natural-language requirements.
4. Run Phase 2 to generate the prototype using sketches and Phase 1 requirements.
5. Save requirements, raw prototype text, rendered HTML/CSS/JavaScript, metadata and logs.
6. Use the reviewer workbook to evaluate generated artifacts.

## Scenario Coverage

The package includes three scenarios:

- E-commerce Checkout System
- Healthcare Appointment System
- Service Portal System

## Model Coverage

The package includes four model folders:

- experiment_GPT4.1
- experiment_GPT4o
- experiment_claude-sonnet-4-5
- Gemini_2.5_Pro
